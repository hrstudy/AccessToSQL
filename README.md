# Access → SQL Server 数据迁移工具

一款基于 Python 开发的桌面端图形化数据迁移工具，旨在提供稳定、高效、可控的 Access 数据库向 SQL Server 迁移方案。有效解决传统导入工具在面对脏数据、自增主键冲突、大数据量中断等问题时的痛点。

## ✨ 核心功能特性

- **🛠️ 智能字段映射**：内置 20+ 种 Access 到 SQL Server 的默认类型映射规则，支持逐表精细化自定义修改每个字段的目标类型。
- **🔑 IDENTITY 自增处理**：全面支持自增字段的创建、清空时重置种子（RESEED）、以及导入时保留原 ID（IDENTITY_INSERT）的特殊处理。
- **💾 断点续传机制**：基于本地 SQLite 记录断点，大数据量导入意外中断后，下次启动可精准从断点位置继续，或选择删除断点重头来过。
- **📊 实时监控与日志**：双进度条（总体表进度 + 当前表行数进度），分级彩色日志实时输出，迁移过程完全透明可控。
- **🛡️ 容错与数据清洗**：自动识别并清洗 Access 历史数据中的隐形空字符（`\x00`）和幽灵空日期（`1899-12-30`），遇到极个别无法转换的脏数据自动跳过并记录详细错误，绝不闪退。
- **📐 自适应窗口**：指定最小窗口尺寸，支持自由拉伸放大以显示更多日志和表格内容。

## 💻 环境要求

> **⚠️ 特别注意（最重要的一步）：**
> 您安装的 **Python 位数** 必须与系统中安装的 **ODBC 驱动位数** 保持完全一致（例如：同为 64 位，或同为 32 位）。否则将无法枚举到任何数据库驱动！

- **Python**: 3.7 及以上版本
- **操作系统**: Windows 7 / 10 / 11
- **依赖库**: `pyodbc`, `PyQt5`
- **系统驱动**: 
  - [Microsoft Access Database Engine](https://www.microsoft.com/en-us/download/details.aspx?id=54920) (用于读取 MDB/ACCDB)
  - [Microsoft ODBC Driver for SQL Server](https://docs.microsoft.com/en-us/sql/connect/odbc/download-odbc-driver-for-sql-server) (用于连接 SQL Server)

## 📦 安装与运行

1. **安装 Python 依赖库**：
   打开命令行（CMD 或 PowerShell），执行以下命令：
   ```bash
   pip install pyodbc PyQt5
   ```

2. **运行程序**：
   将下载的 `accesstosql.py` 文件保存到本地，在命令行中进入该目录并运行：
   ```bash
   python accesstosql.py
   ```

## 📖 使用指南

程序界面分为 5 个标签页，按照从左到右的顺序操作即可：

### ① 数据源配置
1. 在下拉框中选择您的 Access ODBC 驱动（推荐优先选择带 `ACE` 字样的驱动）。
2. 点击“浏览”选择您的 `.mdb` 或 `.accdb` 文件。
3. 点击“连接并加载表列表”，成功后下方会显示该数据库中的所有用户表。

### ② 目标配置
1. 选择 SQL Server 的 ODBC 驱动（如 `ODBC Driver 17 for SQL Server`）。*(注：下拉框支持手动输入)*
2. 填写 SQL Server 的服务器地址、数据库名称。
3. 选择身份验证方式（Windows 身份验证 或 SQL Server 账号密码）。
4. 点击“测试连接”，确保配置正确。

### ③ 表映射配置
1. 勾选您需要迁移的表（支持全选/全不选）。
2. 选择“表已存在时”的处理策略：跳过、清空后重新导入、或追加数据。
3. 配置自增选项（是否保留原ID、清空时是否重置自增种子）。
4. 点击 **“使用默认类型映射”** 快速配置，或点击 **“自定义字段映射”** 逐表精细调整（在弹窗中可看到当前表名，并为每个字段指定目标类型及是否自增）。

### ④ 数据迁移
1. 查看断点信息（如果有历史中断记录，可选择继续或删除）。
2. 点击 **“▶ 开始迁移”**。
3. 观察双进度条和底部日志。如需中止，点击 **“■ 停止迁移”**（断点会自动保存）。

## ❓ 常见问题排查 (FAQ)

**Q: 启动后，Access 驱动或 SQL Server 驱动下拉框是空的？**
A: 这是 ODBC 驱动位数与 Python 位数不一致导致的。请确认您的 Python 是 64 位还是 32 位（在 CMD 输入 `python` 查看），然后去微软官网下载对应位数的 Access Database Engine 重新安装。

**Q: 提示 `表没有标识属性。无法执行 SET 操作 (8106)`？**
A: 已在代码中内置智能检测修复。如果仍然出现，请检查“③ 表映射配置”中，该表的自增字段是否被错误地勾选了"自增"。如果目标表本身没有自增列，请取消勾选。

**Q: 提示 `当 IDENTITY_INSERT 设置为 OFF 时，不能为表中的标识列插入显式值 (544)`？**
A: 已在代码底层修复。程序现在会智能判断：如果不勾选"保留ID"，会自动在生成的 SQL 语句中剔除自增列，让 SQL Server 自动生成 ID。

**Q: 日志显示某张表的数据全部失败跳过？**
A: 通常是原 Access 表中存在"脏数据"。程序已内置自动清洗机制（清理 `\x00` 空字符和 `1899-12-30` 幽灵日期）。如果仍有跳过，请仔细查看日志中红色的 `首条失败原因` 提示，根据具体的 SQL Server 报错调整字段映射类型（如将 `NVARCHAR(255)` 改为 `NVARCHAR(MAX)` 或 `DATETIME2`）。
